# カテゴリ1: ヒューリスティック（経験則）

## 1. ヒックの法則 (Hick's Law)

- **定義**: 選択肢の数が増えると、意思決定に要する時間は対数的に増加する。
- **原著**: Hick, W.E. (1952). *Quarterly Journal of Experimental Psychology.*
- **なぜ効くか**: ワーキングメモリの容量制限により、選択肢が多いほど比較・評価に認知資源を消費し、決定疲れやページ離脱を招く。

### 実装指針
- メニュー・設定画面の選択肢は **5±2個** 以内に収める
- 多数の機能がある場合は段階的開示（Progressive Disclosure）でグループ化する
- チャットUIでは、次に取るべきアクションを1〜3個に絞って提示する
- 「すべて表示」のような逃げ道を用意しつつ、デフォルトでは最小限を見せる

```
// NG: 10個のボタンを一度に表示
// OK: 主要3アクション + 「その他」メニュー
```

## 2. フィッツの法則 (Fitts's Law)

- **定義**: ターゲットの取得時間は、ターゲットまでの距離とサイズの関数である。大きく近いほど速い。
- **原著**: Fitts, P.M. (1954). *Journal of Experimental Psychology.*
- **なぜ効くか**: 運動制御の物理法則に基づく。クリック・タップの精度と速度はターゲットの面積と距離に直接依存する。

### 実装指針
- CTAボタンは十分な大きさ（最小44×44px、推奨48×48px）にする
- 頻繁に使う送信ボタンはフォームの直近に配置する
- 破壊的操作（削除等）のボタンは小さく、主要操作から距離を取る
- モバイルでは親指が届きやすい画面下部にインタラクティブ要素を配置する

---

## 3. ヤコブの法則 (Jakob's Law)

- **定義**: ユーザーは大半の時間を「他のサイト」で過ごしている。あなたのサイトも他のサイトと同じように動作することを期待する。
- **提唱者**: Jakob Nielsen（Nielsen Norman Group）
- **なぜ効くか**: 既存のメンタルモデルを活用することで学習コストをゼロに近づけ、認知負荷を最小化できる。

### 実装指針
- チャット入力は画面下部固定、メッセージは時系列で上から下へ
- ナビゲーション、検索、設定アイコンは業界標準の配置を踏襲する
- 独自UIパターンを導入する場合は、必ずA/Bテストで既存パターンと比較する
- 「革新」より「慣れ」を優先する。ユーザーの期待を裏切る独自性はコストが高い

---

## 4. ミラーの法則 (Miller's Law)

- **定義**: 人間が短期記憶に保持できる情報の塊は平均7±2個である。
- **原著**: Miller, G.A. (1956). *Psychological Review.*
- **なぜ効くか**: ワーキングメモリの容量制限は生物学的制約。これを超える情報提示は処理落ちを引き起こす。

### 実装指針
- 一画面に表示する情報グループは **5〜9個** 以内にする
- 長いリストはチャンキング（意味のある塊にグループ化）して提示する
- ステップが多い操作はウィザード形式に分割し、各ステップの項目数を制限する
- 会話UIでは、一度に提示する選択肢や情報ポイントを4個以下に抑える

---

## 5. テスラーの法則（複雑性保存の法則）(Tesler's Law)

- **定義**: あらゆるシステムには、それ以上削減できない一定量の複雑性が存在する。
- **提唱者**: Larry Tesler
- **なぜ効くか**: 複雑性は消滅しない。ユーザー側の複雑性を減らすには、システム側がその複雑性を吸収する必要がある。

### 実装指針
- ユーザーに選択や設定を強いる前に、スマートデフォルトを設定する
- 「自動検出」「推奨設定」で、ユーザーの意思決定負荷を肩代わりする
- 複雑な機能は段階的に公開し、初期状態はシンプルに保つ
- 避けるべき: 「カスタマイズ可能」を売りにして、全ての設定をユーザーに委ねること
