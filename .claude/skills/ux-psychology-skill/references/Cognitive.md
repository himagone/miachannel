# カテゴリ5: 認知負荷管理

## 24. 認知負荷理論 (Cognitive Load Theory)

- **定義**: ワーキングメモリの容量は限られており、情報処理の負荷が3種類に分かれる。
- **原著**: Sweller, J. (1988). *Cognitive Science.*
- **3種類の認知負荷**:
  - **内在的負荷 (Intrinsic)**: タスク自体の本質的な複雑さ。タスクを変えない限り削減不可。
  - **外在的負荷 (Extraneous)**: 情報の提示方法に起因する不要な負荷。デザインで削減可能。
  - **本質的負荷 (Germane)**: 学習やスキーマ構築に使われる有益な負荷。

### 実装指針（外在的負荷の削減に集中する）
- 視覚的ノイズを排除する: 装飾的な要素、意味のないアニメーション、冗長なテキストを削る
- 一画面一目的の原則: 各画面で達成すべき目的を1つに絞る
- 情報のチャンキング: 長い情報を意味のある塊に分割して段階的に提示する
- スマートデフォルト: ユーザーに考えさせる前に、妥当な初期値を設定する
- 視覚的階層: サイズ・色・配置で情報の優先度を直感的に伝える（→タイポグラフィskillも参照）
- 一貫性: 同じ操作には同じビジュアルパターンを使い、学習転移を促進する

## 25. ポステルの法則 (Postel's Law / Robustness Principle)

- **定義**: 送信するものは厳格に、受信するものは寛容に。
- **原著**: Postel, J. (1980). RFC 761.
- **なぜ効くか**: もともとネットワークプロトコルの原則だが、UIに適用すると「ユーザーの入力には寛容に、出力は正確に」という設計哲学になる。

### 実装指針
- フォーム入力で柔軟なフォーマットを受け入れる（日付、電話番号、全角/半角）
- エラーメッセージは何が間違っているかと、どう修正すればよいかの両方を明示する
- チャット入力では、タイポ・表記揺れに対して寛容に解釈する
- 出力（AI応答）は構造化され、正確で、一貫した形式にする
