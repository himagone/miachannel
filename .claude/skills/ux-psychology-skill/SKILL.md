# UX心理学リファレンス

> UIUXの設計・改善時に参照するUX心理学原則ガイドライン。
> 出典: Jon Yablonski『Laws of UX』、Nielsen Norman Group、各認知心理学の原著論文を基盤とする。

---

## 使い方

このスキルはSKILL.md（このファイル）とreferences/配下のカテゴリ別ファイルで構成されている。

**毎回全ファイルを読む必要はない。** 以下のルーティングテーブルを使って、タスクに関連するファイルだけを読む。

---

## ルーティングテーブル

タスクの内容に応じて、該当するreferencesファイルを読む。複数該当する場合は複数読む。

| タスク内容 | 読むファイル |
|---|---|
| ボタン・CTA・クリック可能要素の設計 | `references/heuristics.md` (フィッツの法則, ヒックの法則) |
| メニュー・ナビゲーション・選択肢の設計 | `references/heuristics.md` (ヒックの法則, ミラーの法則) |
| フォーム・入力UI・バリデーション | `references/cognitive-load.md` (認知負荷理論, ポステルの法則) |
| カード・リスト・情報表示のレイアウト | `references/gestalt.md` (近接, 類同, 共通運命) |
| アニメーション・トランジション・状態変化 | `references/gestalt.md` (共通運命) + `references/attention.md` (変化盲) |
| エラー表示・フィードバック・通知 | `references/attention.md` (選択的注意, 変化盲) + `references/cognitive-load.md` (ポステルの法則) |
| CTA文言・マイクロコピーの最適化 | `references/bias.md` (フレーミング効果, フォン・レストルフ効果) |
| オンボーディング・ウィザード・ステップUI | `references/motivation.md` (ツァイガルニク効果, 目標勾配効果) + `references/heuristics.md` (ミラーの法則) |
| 料金ページ・プラン比較 | `references/bias.md` (アンカリング, デコイ効果, 損失回避) |
| 解約・退会フロー | `references/bias.md` (損失回避, 保有効果) + `references/motivation.md` (IKEA効果) |
| チャットUI全般 | `references/heuristics.md` (ヤコブの法則, ヒックの法則) + `references/cognitive-load.md` |
| ユーザーの定着・リテンション施策 | `references/motivation.md` (全般) + `references/bias.md` (社会的証明) |
| 全般的なUI改善・レビュー | `references/heuristics.md` + `references/cognitive-load.md` を優先的に読む |

**判断に迷ったら**: `references/heuristics.md` と `references/cognitive-load.md` の2つを読めば、大半のUI実装タスクをカバーできる。

## 原則一覧（クイックリファレンス）

参照先を素早く特定するための全原則インデックス。

### heuristics.md（経験則）
1. ヒックの法則 — 選択肢が多いほど決定が遅くなる
2. フィッツの法則 — 大きく近いターゲットほど速く操作できる
3. ヤコブの法則 — ユーザーは他サイトと同じ動作を期待する
4. ミラーの法則 — 短期記憶は7±2チャンク
5. テスラーの法則 — 複雑性は消えない、システムが吸収すべき

### gestalt.md（ゲシュタルト原則）
6. 近接の法則 — 近い要素は関連して見える
7. 類同の法則 — 似た見た目の要素は同グループに見える
8. 共通運命の法則 — 同方向に動く要素は関連して見える

### bias.md（認知バイアス・応用原則）
9. フォン・レストルフ効果 — 異質なものが記憶に残る
10. 系列位置効果 — リストの先頭と末尾が記憶に残る
11. ピーク・エンドの法則 — 体験のピークと終了時で評価が決まる
12. アンカリングバイアス — 最初の情報が基準になる
13. 損失回避 — 損失は利得の2倍痛い
14. 社会的証明 — 他者の行動を参考にする
15. 選択のパラドックス — 選択肢が多すぎると満足度が下がる
16. デコイ効果 — おとりの選択肢でターゲットの選択率が上がる
17. フレーミング効果 — 提示の仕方で判断が変わる
18. 希少性バイアス — 限定されたものに高い価値を感じる
19. パーキンソンの法則 — 時間は与えられただけ使い切る

### motivation.md（動機づけ・行動設計）
20. ツァイガルニク効果 — 未完了タスクは記憶に残りやすい
21. 目標勾配効果 — ゴールに近づくほど加速する
22. IKEA効果 — 自分で作ったものに高い価値を感じる
23. 保有効果 — 所有物に高い価値を認める

### cognitive-load.md（認知負荷管理）
24. 認知負荷理論 — 外在的負荷をデザインで削減する
25. ポステルの法則 — 入力に寛容、出力に厳格

### attention.md（注意と記憶）
26. 選択的注意 — 目標関連の刺激のみに注意が向く
27. 変化盲 — 注意外の変化は気づかれない

## 既に適用済みの原則

重複実装を避けるために記載。新規プロジェクトでは無視して良い。

| 原則 | 適用内容 |
|---|---|
| 美的ユーザビリティ効果 | shadcn/ui + Tailwind CSS による統一カラーパレットと余白 |
| 労働の錯覚 | スピナー付き「考え中...」表示 |
| ドハティの閾値 | ストリーミングレスポンスによる即時フィードバック |
| 親近性バイアス | 吹き出しデザイン・下部固定フォームの採用 |

## 倫理ガイドライン

UX心理学の適用において遵守する原則。

1. **透明性**: ユーザーの行動を操作するのではなく、目標達成を支援する目的で使用する
2. **誠実性**: 虚偽の希少性、偽のカウントダウン、水増しした数値は使用しない
3. **自律性**: ユーザーが十分な情報に基づいて判断できるようにする。ダークパターンを排除する
4. **可逆性**: ユーザーの行動は容易に取り消せるようにする
5. **包括性**: 認知的多様性（ADHD、高齢者、非ネイティブ等）を考慮する
